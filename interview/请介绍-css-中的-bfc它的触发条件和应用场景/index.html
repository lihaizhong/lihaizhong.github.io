请介绍 CSS 中的 BFC，它的触发条件和应用场景

Jul 13, 2025
查看答案

BFC（Block Formatting Context, 块级格式化上下文）是 CSS 中一种特殊的渲染环境，它就像一个容器的“容器”，容器内部的元素布局不会影响到外部元素，外部元素的布局也不会干扰容器内部。

触发条件

  • 根元素（<html>）
  • 浮动元素（float 值为 left 或 right，且不为 none）
  • 绝对定位元素（position 值为 absolute 或 fixed）
  • 行内块元素（display: inline-block）
  • 表格单元格（display: table-cell，默认属性）
  • 表格标题（display: table-caption，默认属性）
  • overflow 值不为 visible 的块元素（如 overflow: hidden、auto、scroll）
  • 弹性元素（display: flex、inline-flex 的直接子元素）
  • 网格元素（display: grid 或 inline-grid 的直接子元素）等

应用场景

解决 margin 重叠问题

当两个相邻的块级元素都设置了 margin 时，可能会出现 margin 重叠（取较大值），将她们放入不同的 BFC 容器中可避免重叠。

清除浮动

父元素内部子元素浮动时，父元素可能会因没有内容支撑而高度坍塌，给父元素触发 BFC 后，父元素会包裹住浮动的子元素，从而清除浮动影响。

阻止元素被浮动元素覆盖

当一个元素浮动时，可能会覆盖旁边的非浮动元素，给非浮动元素触发 BFC，可使它不被覆盖，保持与浮动元素并列的布局。
